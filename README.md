# LanPlay PoC - UDP Interception via Fake Gateway

This project is a Proof of Concept (PoC) for Android 13+ designed to intercept UDP traffic from a Nintendo Switch. It demonstrates how to use Android's `VpnService` to act as a "Fake Gateway" for a specific static subnet (`10.13.0.0/16`), allowing the capture of LAN Play packets without requiring root access.

## 🚀 Architecture (How it Works)

The solution overcomes the limitation where a Nintendo Switch configured with a static IP cannot communicate with the Android Hotspot's default gateway because they are on different subnets.

1.  **Hotspot Layer (L2):** Uses Android's `LocalOnlyHotspot` to create a Wi-Fi network. This provides the physical link between the Switch and the Android device.
2.  **VPN Layer (L3 - The Interceptor):** A `VpnService` creates a virtual network interface (`tun0`).
    -   **IP Address:** The VPN interface is assigned `10.13.37.1`. This is the IP the Switch expects as its Gateway.
    -   **Routing:** We add a route for `10.13.0.0/16` to the VPN. This tells the Android kernel: "Any traffic destined for the 10.13.x.x network should be sent to this VPN app."
3.  **Fake Gateway:** When the Switch sends a packet to its configured gateway (`10.13.37.1`), the Android kernel routes it into our `VpnService`.
4.  **Raw Sniffing:** The app reads raw bytes from the VPN's file descriptor, manually parses the IP and UDP headers, and logs the captured traffic.

## 🛠 Setup Instructions

### 1. Android Device
1.  Launch the **LanPlayPoC** app.
2.  Tap **START ALL**.
3.  Grant the requested permissions:
    -   **Nearby Devices / Location:** Required for Hotspot management.
    -   **VPN Connection:** Required to intercept the traffic.
4.  Once started, check the logs in the app to find the **Hotspot SSID** and **Password**.

### 2. Nintendo Switch
Configure the connection manually to match the "Fake Gateway" environment:

1.  Go to **System Settings** > **Internet** > **Internet Settings**.
2.  Select the SSID generated by the Android app.
3.  Select **Change Settings**.
4.  **IP Address Settings:** Manual
    -   **IP Address:** `10.13.0.100`
    -   **Subnet Mask:** `255.255.0.0`
    -   **Gateway:** `10.13.37.1`
5.  **DNS Settings:** Manual
    -   **Primary DNS:** `8.8.8.8`
    -   **Secondary DNS:** `8.8.4.4`
6.  **Save** and connect to the network.

## 🧪 How to Test

1.  Ensure the Switch is connected to the Hotspot and configured with the manual IP settings above.
2.  On the Switch, start a game that supports LAN Play (e.g., *Mario Kart 8 Deluxe*, *Splatoon 2*).
3.  Enter the LAN Play menu in the game (usually by holding L+R+L3 in the Local Wireless menu).
4.  Search for or create a room.

### ✅ Success Criteria
The Android app's log area should display:
`[SUCCESS] Captured UDP from 10.13.0.100:XXXX to 10.13.37.1:11451`

This confirms that the packet traveled from the Switch, through the Wi-Fi interface, was routed by the Kernel into our VPN, and successfully parsed by our app.

## 🔍 Troubleshooting

*   **No UDP Logs:** Check if the app logs show any "DHCP Discovery" activity (ports 67/68). If they do, the Switch is likely still set to "Automatic" IP settings.
*   **VPN Disconnects:** Ensure "Always-on VPN" is not interfering in Android System Settings.
*   **Connection Test Fails on Switch:** This is expected! The "Fake Gateway" only intercepts specific LAN Play traffic; it does not provide full internet access or a real gateway response to the Switch's connectivity check. The "Success" is the UDP capture log in the app.
