# Design Spec - Restore Real Hotspot Info

## 1. Overview
The user wants to see both the target credentials (SSID: `Switch-Lan`, Pass: `12345678`) and the actual credentials generated by Android in the `HotspotManager` status log.

## 2. Components

### 2.1 HotspotManager.kt
- **SSID/Password Extraction:** Re-introduce the logic to extract the actual SSID and password from the `WifiManager.LocalOnlyHotspotReservation` object.
    - For Android R (API 30) and above, use `res.softApConfiguration`.
    - For older versions, use `res.wifiConfiguration`.
- **Status Logging:** Update `onStatus` to show:
    ```
    Hotspot Started
    Target SSID: Switch-Lan
    Target Pass: 12345678
    Actual SSID: <real_ssid>
    Actual Pass: <real_pass>
    ```

## 3. Data Flow
1. `HotspotManager.start()` is called.
2. `onStarted(res)` is triggered by the system.
3. Actual SSID and password are extracted from `res`.
4. `onStatus` is called with the formatted string containing both target and actual credentials.
